Some bad coding practices I actually encountered in production.

Exec Misuse

I know that the __import__ mechanism uses "exec" but this is quite confusing, especially if you have access only to "main.py"

why ? Because "iris_data" was frequently used in different scripts and this was the adopted solution to reduce code duplication

and multiply headaches

main.py


import pandas as pd


exec(open("path/to/my_module.py"))

# where does iris_data come from ?
result = pd.mean(iris_data)

my_module.py


# oh, here it is !
iris_data = pd.read_csv("path/to/iris.csv")

import Misuse

slight improvement on the previous version

I want to stress that every time you import the module, the code in it will run and these are not light operations (usually, they were DB calls !).

Even better: if the import system caches the imported value, the Heisenbug risk skyrockets !

main.py


import pandas as pd

import my_module


result = pd.mean(my_module.iris_data)

my_module.py


iris_data = pd.read_csv("path/to/iris.csv")
